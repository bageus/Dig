# Туман войны, разведка и источники обзора

Статус: authoritative design. Design-задача: #147. Реализация: #165.

## Назначение

Система отделяет знание игрока о карте от текущей видимости. World/Exploration является единственным владельцем gameplay visibility; Unity только отображает snapshot.

## Состояния клетки

Каждая клетка имеет одно из трёх производных состояний:

- `Unexplored` — клетка никогда не была видима; отображается чёрным туманом, содержимое неизвестно;
- `ExploredNotVisible` — клетка была разведана, но сейчас не покрыта источником обзора;
- `Visible` — клетка сейчас покрыта хотя бы одним действующим источником обзора.

Признак `Explored` сохраняется после ухода или удаления источника. При потере обзора клетка переходит `Visible -> ExploredNotVisible`, а не обратно в `Unexplored`.

## Источники обзора

| Источник | Радиус |
|---|---:|
| Гном | 10 |
| Обычное исправное здание | 10 |
| Повреждённое здание | 2 |
| Лестница | 2 |
| Лифт | 2 вдоль всей шахты |
| Дверь | 2 |
| Ловушка | 2 |
| Могила | 5 |
| Предмет на земле | 0 |

Радиус обычного здания считается от ближайшей внешней клетки его footprint: итоговая зона является объединением зон от граничных клеток здания.

Повреждённым для vision считается здание, перешедшее в повреждённый runtime-state согласно Building durability policy. Точный health threshold хранится в BuildingDefinition/repair policy и не дублируется в Exploration.

## Геометрия обзора

Обзор распространяется по открытому объёму мира, а не через сферу, игнорирующую стены.

Для детерминированной реализации используется flood-fill по 6-связному графу соседних клеток `±X`, `±Y`, `±Z` с ограничением graph distance радиусом источника.

Блокируют распространение:

- сплошная порода;
- стены и непроходимые строительные клетки;
- целый потолок между пещерами;
- закрытая дверь.

Разрешают распространение:

- выкопанный проход;
- открытый дверной проём;
- лестничная связь;
- шахта лифта по всей её длине.

Следствия:

- пещера над другой пещерой не раскрывается через целый потолок, даже если геометрический радиус достаточен;
- соседнее помещение без прохода не раскрывается;
- помещение с открытым проходом раскрывается в пределах оставшегося graph distance;
- обзор не перескакивает между Z-слоями без физически открытой связи.

## Lifecycle источника

- установка/появление источника пересчитывает только затронутые chunks;
- перемещение гнома обновляет dirty region инкрементально;
- удаление, упаковка или уничтожение здания снимает его текущий обзор;
- повреждение здания изменяет радиус на 2 и обновляет visibility;
- ремонт возвращает радиус definition;
- закрытие/открытие двери инвалидирует локальную visibility graph;
- Presentation lights не являются источником обзора.

## Предметы и память карты

Предметы не раскрывают клетки.

Когда предмет был видим, карта может хранить `LastKnownWorldItemMarker` с ItemId, последней известной клеткой и tick наблюдения. После потери видимости marker остаётся в разведанной области. Если предмет переместился вне обзора, marker не обновляется до следующего наблюдения и может быть устаревшим.

Gameplay-системы не должны считать remembered marker фактическим доступным ItemStack. Hauling, pickup, targeting и reservations используют только authoritative item state плюс требуемую текущую visibility policy.

## Визуальное освещение

При установке здания Presentation может добавлять:

- внутреннее освещение здания;
- светильники на стенах вокруг него;
- визуальное затухание в `ExploredNotVisible`.

Светильники являются rebuildable view и не расширяют gameplay radius сами по себе.

## Интеграция

- World хранит terrain и blockers;
- Exploration хранит persistent explored и текущие source contributions;
- Buildings/Agents публикуют source snapshots;
- Doors предоставляют authoritative open/closed state;
- Inventory предоставляет observed-item events, но не visibility;
- Jobs/Hauling/Combat запрашивают visibility через узкий read contract;
- Presentation различает три состояния не только цветом, но также opacity/pattern/label в accessibility mode.

## Save/Load

Сохраняются:

- persistent explored bitset/version по chunks;
- last-known item markers, если включены в map-memory profile;
- данные, необходимые для восстановления source identity/version.

Current `Visible` после загрузки пересчитывается из authoritative residents/buildings/doors, а не доверяет устаревшей сериализованной копии.

## Инварианты

- `Visible` всегда означает `Explored`;
- источник не видит через целую стену или потолок;
- удаление источника не стирает историю разведки;
- предмет не становится gameplay-доступным только из-за last-known marker;
- одинаковый world/source snapshot даёт одинаковую visibility mask;
- UI не изменяет ExplorationState.

## Проверки

Обязательные тесты:

- resident radius 10;
- standard/damaged building 10/2;
- ladder, door, trap radius 2;
- grave radius 5;
- lift coverage вдоль шахты;
- потолок между Z-слоями;
- соседние комнаты с проходом и без прохода;
- закрытая и открытая дверь;
- source removal `Visible -> ExploredNotVisible`;
- stale last-known item marker;
- hauling source visibility;
- Save/Load и deterministic replay;
- dirty-chunk performance budget.
