# Головные уборы по текущей деятельности гнома

Связанная задача: #151.

## Назначение

Головной убор является визуальным индикатором последней или текущей рабочей роли гнома. Authoritative состояние хранится как typed appearance/read-model data; Unity только отображает snapshot.

## Таблица ролей

| Состояние/работа | Головной убор |
|---|---|
| Свободное время | обычный колпак |
| Рабочее время без занятия | кепка козырьком назад |
| Копание, каменное дело, работа с металлом | шахтёрская каска с фонарём |
| Доставка, упаковка, сборка здания | фуражка |
| Рубка грибов/работа с древесиной | защитные наушники |
| Исследование или обучение | колпак со звёздами |
| Приготовление еды, зелий или алкоголя | колпак повара |
| Лечение | медицинская шапочка |
| Сражение или охрана | воинский шлем |

## Lifecycle смены

- смена роли не должна мгновенно заменять модель: существует короткая переходная фаза `NoHeadwear`;
- движение к точке само по себе не меняет рабочий головной убор;
- если шахтёр получил команду идти, он остаётся в шахтёрской каске до назначения другой роли;
- перед боевой атакой гном сначала переходит к воинскому шлему и только затем начинает combat action;
- переход в свободное время назначает обычный колпак;
- безделье определяется только как отсутствие работы в рабочее время и назначает кепку козырьком назад.

## Модель данных

```text
ResidentRoleVisualState
- ResidentId
- ActiveRoleVisualId
- PendingRoleVisualId?
- TransitionState
- TransitionProgress
- SourceActionId?
- Version
```

Display name статуса не используется для выбора модели. Mapping выполняется по typed action/role category.

## Save/Load и Presentation

Сохраняются активная и ожидаемая роль, переход и source action. После загрузки world renderer и HUD portrait получают один `ResidentAppearanceSnapshot`.
