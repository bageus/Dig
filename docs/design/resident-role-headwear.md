# Головные уборы по текущей деятельности гнома

Связанная задача: #151.

## Назначение

Головной убор является только визуальным индикатором текущей или последней рабочей роли гнома. Он не увеличивает обзор, не добавляет защиту и не меняет скорость или эффективность работы.

Authoritative состояние хранится как typed appearance state; Unity отображает snapshot и не выбирает роль самостоятельно.

## Постоянная внешность

- одежда гнома постоянна;
- обычная причёска постоянна до перехода в старость;
- у старого гнома волосы становятся седыми;
- у старого мужчины дополнительно появляется лысина;
- рабочая деятельность меняет только головной убор.

## Таблица ролей

| Состояние/работа | Головной убор |
|---|---|
| Свободное время | обычный колпак |
| Рабочее время без занятия | кепка козырьком назад |
| Копание, каменное дело, работа с металлом | шахтёрская каска с фонарём |
| Доставка, упаковка, сборка здания | фуражка |
| Рубка грибов/работа с древесиной | защитные наушники |
| Исследование или обучение | колпак со звёздами |
| Приготовление еды, зелий или алкоголя | колпак повара |
| Лечение | медицинская шапочка |
| Сражение или охрана | воинский шлем |

Для смешанной работы используется явная primary role действия. Подтверждённое правило: доставка любого материала, включая металл, использует фуражку, а не шахтёрскую каску.

## Lifecycle смены

- перед началом любой работы система проверяет требуемый головной убор;
- если убор не совпадает, сначала выполняется короткий переход переодевания через `NoHeadwear`;
- только после завершения перехода начинается gameplay action;
- если нужный убор уже надет, работа начинается без дополнительного перехода;
- движение к точке само по себе не меняет рабочий головной убор;
- шахтёр, которому приказали идти, остаётся в каске до назначения другой роли;
- перед атакой обязательно завершается переход к воинскому шлему;
- свободное время назначает обычный колпак;
- Work schedule без работы назначает кепку козырьком назад.

Точная продолжительность переодевания является data-driven balance value из Q-014.

## Колпак конкретного гнома

После смерти создаётся отдельный identity-linked предмет `Колпак {ИмяГнома}`. Это не текущая рабочая шапка и не копия каски/фуражки/шлема. Предмет всегда символизирует конкретный `ResidentId` и используется системой могил и возвращения жителя.

## Модель данных

```text
ResidentAppearanceState
- ResidentId
- ClothingAppearanceId
- HairAppearanceId
- AgeAppearanceVariant
- ActiveHeadwearRoleId
- PendingHeadwearRoleId?
- TransitionState
- TransitionProgress
- SourceActionId?
- Version
```

Display name статуса не используется для выбора модели. Каждое action definition предоставляет явный `HeadwearRoleId`.

## Save/Load и Presentation

Сохраняются постоянная одежда, возрастной вариант причёски, активная и ожидаемая роль, переход и source action. World renderer и HUD portrait используют один `ResidentAppearanceSnapshot`.

## Инварианты

- головные уборы не дают gameplay modifiers;
- работа не начинается до подтверждения нужного визуального состояния;
- команда движения не сбрасывает last work role;
- identity cap после смерти не зависит от надетой рабочей шапки;
- старение меняет волосы, но не identity или одежду;
- missing asset использует диагностируемый fallback.
