# Доступность, длительность и UI исследования

Связанная задача: #128. Content graph: #126.

## Доступность

Технология не изучается автоматически при росте навыков.

1. Система проверяет всех живых гномов.
2. Должен существовать хотя бы один гном, одновременно удовлетворяющий всем skill requirements.
3. Игрок выбирает здание-владельца технологии.
4. Нажатие на иконку создаёт Research job.
5. Выполнять job может только гном, удовлетворяющий требованиям этой технологии.
6. Исследователь выполняет работу в своё рабочее время.

## Состояния иконки

- `Locked` — оранжевая: ни один живой гном не достигает требований;
- `EligibleAvailable` — белая: требования достигнуты и существует подходящий исследователь в рабочем времени;
- `EligibleResting` — жёлтая: подходящий исследователь существует, но сейчас отдыхает;
- `InProgress` — отдельный progress state;
- `Completed/Producible` — технология изучена и запись становится производимой.

Цвет всегда дублируется текстом, иконкой состояния и списком требований.

## Длительность

Базовая длительность выводится из recipe производства одного цикла целевого предмета или BuildingBox.

```text
ResearchDuration = Σ (IngredientQuantity × IngredientResearchMinutes)
```

Начальная таблица:

| Ингредиент | Минут за единицу |
|---|---:|
| Шляпка гриба | 1 |
| Ножка гриба | 1 |
| Хомяк | 1 |
| Камень | 1 |
| Личинка | 1 |
| Железо/железный слиток | 2 |
| Золото | 4 |
| Кристалл | 5 |

Пример: recipe из четырёх ножек даёт 4 игровые минуты исследования.

Таблица data-driven и использует stable ItemId. Display name не участвует в расчёте.

## Research job

- click не завершает технологию, а создаёт одну idempotent job;
- workshop reservation и researcher reservation атомарны;
- progress сохраняется при Save/Load;
- завершение публикует TechnologyCompleted один раз;
- после завершения верхняя строка уведомлений показывает `{Название технологии} изобретено`;
- completed entry показывает состав материалов для производства и доступные production commands.

## Владение состоянием

Technology хранит Locked/Eligible/InProgress/Completed и progress. Skills предоставляет authoritative requirements query. Jobs владеет researcher/workshop reservation. Presentation только отправляет StartResearchCommand и отображает read model.

## Подтверждённое решение Q-035

Квалифицированный гном обязан лично выполнять исследование. Неквалифицированный гном не может заменить его только потому, что технология уже стала Eligible.
