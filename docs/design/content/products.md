# Продукция и производственные выходы

Этот файл перечисляет предметы, которые создаются Production system. Рецепты с конкретными эффектами алкоголя дополнительно описаны в [`alcohol.md`](alcohol.md).

## Общие правила

- каждый продукт имеет стабильный `ItemId`;
- output создаётся только после успешного завершения production order;
- входные ресурсы резервируются и списываются транзакционно;
- один заказ не может выдать результат дважды;
- количество выхода задаётся в `RecipeDefinition`;
- предметы-комплекты зданий имеют `MaxStackSize` и правила хранения как обычные ItemStack;
- отображаемое название не используется как ссылка.

## Единицы продукции

- бочонок и бутылка являются одной единицей `ItemStack.Quantity`;
- одна порция употребления расходует одну единицу соответствующего продукта;
- тара не является вложенным контейнером;
- визуальная тара может отличаться, но не создаёт дополнительное authoritative состояние.

## Продукция зданий-сервисов

Игровая комната, кинотеатр, бар и университет не создают физические ItemStack за завершённый сервисный цикл. Они создают доменные эффекты:

- игровая активность — изменение настроения и разрешение превысить mood cap;
- киносеанс — настроение и временный fertility modifier;
- обслуживание бара — настроение, сон для крепких напитков и опыт `Услуги`;
- университетский цикл — увеличение `TotalSkillCapacity` и опыт `Услуги` работника.

Такие эффекты публикуются только после успешного завершения действия и не должны моделироваться как фиктивные предметы.

## Будущее дополнение

При добавлении нового продукта требуется указать:

- stable ItemId;
- категории;
- максимальный stack size;
- производящее здание;
- recipe inputs/outputs;
- требуемый навык и выдаваемый опыт;
- прямой use effect;
- service-specific effect, если он отличается;
- visual definition;
- technology prerequisites.
