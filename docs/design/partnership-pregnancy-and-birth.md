# Партнёрство, беременность и рождение

Связанные задачи: #11, #145 и #150.

## Active pair

- пара создаётся только между мужчиной и женщиной;
- она эксклюзивна: у resident не больше одной active pair;
- запрещённое родство исключает кандидата;
- главный положительный фактор — совпадение расписаний;
- выбор взаимный, stable `ResidentId` используется как tie-break;
- смерть одного партнёра закрывает active pair;
- переживший партнёр становится свободным и может создать новую пару;
- закрытая связь сохраняется как historical partnership/family relation.

## Условия встречи

Оба партнёра должны иметь Mood `>75`, Nutrition `>80`, Alertness `>80`, Health `=100`, находиться в свободном времени, быть живыми, нестарыми и достижимыми друг для друга.

Система резервирует свободную достижимую клетку для совместной встречи. При успешном commit зачатие гарантировано. Прерывание до commit освобождает reservation.

## Беременность и рождение

- длительность — 1 игровой день;
- parent links фиксируются при commit встречи;
- рождается один ребёнок;
- лимита детей у пары нет;
- после родов действует cooldown 2 игровых дня.

## Омоложение во время беременности

Зелье омоложения из #150 разрешено беременной resident.

При commit употребления:

- pregnancy state очищается атомарно;
- ребёнок не создаётся;
- запланированный birth transition отменяется;
- postpartum cooldown не запускается, поскольку родов не было;
- после нового взросления resident снова участвует в reproduction по обычным eligibility rules.

Save/Load не должен восстанавливать отменённую pregnancy или повторно применять cancellation.

## Возвращение прежнего партнёра

Return восстанавливает identity и historical relations, но не отменяет события после смерти.

Если переживший партнёр уже создал новую пару:

- новая active pair сохраняется;
- старая связь остаётся historical;
- возвращённый resident не добавляется во вторую active pair;
- новая пара автоматически не разрывается;
- старая пара может появиться снова только через обычный candidate-scoring, если в будущем оба бывших партнёра одновременно свободны;
- даже когда оба свободны, return сам по себе не восстанавливает пару автоматически.

## Владение и сохранение

Society владеет active/historical links и matching. Lifecycle владеет birth/death/return и pregnancy cancellation. Needs предоставляет eligibility. Reservations владеет встречей. Presentation только отображает состояние.

Save хранит active pair отдельно от historical relations. Load не превращает historical relation в active pair и не восстанавливает отменённую беременность.

## Критерии приёмки

- active pair всегда эксклюзивна;
- смерть освобождает пережившего партнёра;
- return не разрывает новую пару и не создаёт вторую;
- former partners могут снова стать парой только обычным способом и только когда оба свободны;
- успешная eligible-встреча гарантирует зачатие;
- беременность длится сутки, cooldown после родов — двое суток;
- rejuvenation беременной отменяет pregnancy без рождения и postpartum cooldown;
- Save/Load не повторяет transitions.
