# Основные игровые системы Dig

## 1. Строительство и экономика

### Building Placement

Обязанности:

- проверка геометрии;
- доступность поверхности;
- проект здания;
- требуемые клетки;
- конфликт с другими объектами;
- создание связанных Work Orders.

### Construction

Владеет runtime-состоянием строительной площадки.

Обязанности:

- доставка материалов;
- этапы строительства;
- рабочие позиции;
- прогресс;
- завершение;
- демонтаж и ремонт.

### Production

Владеет производственными заказами и очередями.

Обязанности:

- рецепты;
- входные ресурсы;
- инструменты;
- навык;
- эффективность;
- выходные предметы;
- остановка при нехватке условий;
- создание работ доставки и выполнения.

### Energy

Обязанности:

- источники и потребители;
- локальные сети;
- доступная мощность;
- отключение по приоритету;
- хранение энергии, если оно войдёт в scope;
- диагностика дефицита.

## 2. Прогрессия и контент

### Skills

Обязанности:

- опыт за практику;
- уровни или непрерывные значения;
- влияние на скорость и качество;
- ограничения действий;
- потенциал развития;
- перенос допустимых значений между стадиями жизни.

### Technology

Владеет состоянием открытий.

Обязанности:

- граф prerequisites;
- открытые определения;
- прогресс исследования или изобретения;
- условия доступности;
- события открытия;
- миграция при изменении контента.

### Content Validation

Обязанности:

- проверка схем;
- уникальность ID;
- валидность ссылок;
- обнаружение циклов технологий;
- корректность рецептов;
- проверка локализации;
- отчёт об ошибках до запуска игры.

## 3. Общество

### Relationships

Владеет социальными связями.

Обязанности:

- симпатия;
- доверие;
- социальная память;
- партнёрства;
- конфликты;
- объяснимые изменения отношений.

### Family and Reproduction

Владеет семейными связями и состоянием беременности.

Обязанности:

- родители и дети;
- ограничения родства;
- выбор партнёра;
- беременность;
- рождение;
- количество детей;
- собственные правила наследования характеристик.

### Lifecycle

Владеет стадией жизни.

Обязанности:

- ребёнок;
- взросление;
- взрослый;
- старость;
- смерть;
- переход между стадиями;
- очистка ссылок после смерти.

### Leisure and Social Activities

Обязанности:

- сон;
- еда;
- разговоры;
- развлечения;
- использование помещений;
- комфорт;
- влияние на настроение и отношения.

## 4. Конфликт

### Threat Detection

Обязанности:

- обнаружение врагов;
- оценка угрозы;
- предупреждения;
- поиск безопасной области;
- выбор реакции;
- публикация данных для стратегического ИИ.

### Combat

Владеет боевыми намерениями и результатами атак.

Обязанности:

- атаки и защита;
- оружие;
- дистанции;
- попадания;
- урон;
- статусы;
- отступление;
- разрушение объектов при необходимости.

### Factions and Diplomacy

Владеет отношениями кланов.

Обязанности:

- территория;
- нарушения;
- отношение к другим фракциям;
- война и перемирие;
- торговля в будущем;
- память о значимых событиях.

### Strategic AI

Обязанности:

- цели клана;
- развитие экономики;
- расширение;
- защита;
- нападение;
- распределение жителей;
- редкий стратегический tick;
- объяснение выбранной стратегии.

## 5. Сохранение и загрузка

### Snapshot Builder

Обязанности:

- сбор только авторитетного состояния;
- стабильный порядок сериализации;
- фиксация версий;
- исключение пересчитываемых кэшей;
- проверка целостности ссылок.

### Save Repository

Обязанности:

- безопасная запись;
- временный файл и атомарная замена;
- чтение метаданных;
- обработка повреждений;
- управление слотами сохранений.

### Save Migration

Обязанности:

- последовательные миграции версий;
- отчёт об изменениях;
- отказ с понятной причиной;
- тесты старых fixtures;
- сохранение стабильных ID контента.

## 6. Presentation

### World View

Обязанности:

- меши чанков;
- визуальные материалы;
- здания;
- предметы;
- выделение клеток;
- эффекты изменений мира.

### Agent View

Обязанности:

- визуальное положение;
- интерполяция;
- анимации;
- экипировка;
- эмоции и статусы;
- создание и удаление visual entity.

### UI and Input

Обязанности:

- преобразование ввода в Application Commands;
- HUD;
- окна управления поселением;
- приоритеты и расписания;
- производственные очереди;
- технологическое дерево;
- уведомления.

## 7. Диагностика и качество

### Diagnostics

Минимальные представления:

- инспектор жителя;
- инспектор работы;
- активные резервирования;
- навигационные регионы;
- dirty-чанки;
- причины блокировок;
- стоимость систем по tick;
- версии snapshots и фоновых результатов.

### Testing

Обязательные наборы:

- unit tests;
- integration tests;
- deterministic simulation scenarios;
- save/load fixtures;
- regression tests;
- property-based generation tests;
- тесты инвариантов.

### Performance

Обязанности:

- бюджеты времени и памяти;
- тесты больших поселений;
- тесты больших карт;
- измерение навигации после массового копания;
- контроль allocations;
- simulation LOD для дальних областей.

### Static Quality Gates

Автоматические проверки:

- ручной файл не превышает 350 строк;
- отсутствуют запрещённые зависимости;
- content IDs уникальны;
- ссылки контента валидны;
- тесты проходят;
- сохранения совместимы с fixtures.
