# Примечания к системным Issues

Этот файл временно фиксирует единый формат задач до добавления GitHub Issue Templates.

Каждый системный Issue должен содержать:

- цель;
- владельца состояния;
- scope;
- зависимости;
- результаты;
- критерии приёмки;
- тестовые сценарии;
- диагностику;
- ограничения;
- ссылку на `docs/development-rules.md`.

После появления `.github/ISSUE_TEMPLATE/` этот файл следует удалить или преобразовать в ссылку на шаблон.
