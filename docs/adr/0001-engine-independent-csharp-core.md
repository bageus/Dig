# ADR-0001: Независимое от движка C#-ядро

- Статус: accepted
- Дата: 2026-07-14
- Владельцы: проект Dig

## Контекст

Проекту нужна реализация доменных правил до окончательного закрепления Unity или Godot. Игровой движок не должен становиться владельцем состояния мира, жителей, работ и экономики. Также требуется возможность запускать симуляционные тесты без графического редактора.

## Рассмотренные варианты

1. Сразу хранить игровую логику в компонентах выбранного движка.
2. Создать независимое C#-ядро и подключать движок через адаптеры.
3. Отложить реализацию до окончательного выбора движка.

## Решение

Создать независимые сборки `Dig.Domain` и `Dig.Application`, на которые не ссылается API игрового движка. Базовые библиотеки ориентируются на `netstandard2.1`, а автоматические тесты запускаются на .NET 8.

`Dig.Infrastructure` реализует технические порты. `Dig.Presentation.Abstractions` содержит только read-model и presenter-контракты. Конкретный Unity- или Godot-слой будет добавлен отдельным адаптером после выбора движка.

Это решение не является окончательным выбором Unity или Godot.

## Последствия

### Положительные

- доменные тесты запускаются без редактора;
- правила симуляции не зависят от сцен и объектов движка;
- движок можно заменить без переписывания доменной модели;
- проще поддерживать один источник истины;
- будущая headless-симуляция не требует графического слоя.

### Отрицательные

- требуется явное преобразование между симуляцией и представлением;
- нельзя использовать типы движка в публичных контрактах ядра;
- необходимо поддерживать архитектурные проверки зависимостей.

## Критерии пересмотра

Решение пересматривается, если выбранный runtime не способен использовать эти C#-сборки или измерения покажут неприемлемую стоимость границы между симуляцией и представлением.
