# Добавление нового модуля

Этот файл дополняет `docs/development-rules.md`, не переопределяя его. Правила разработки и исключения фиксируются только там.

## До написания кода

Зафиксировать:

- ответственность модуля;
- авторитетного владельца изменяемого состояния;
- входящие команды;
- read-only запросы;
- публикуемые события-факты;
- инварианты;
- разрешённые зависимости;
- формат snapshot или способ полного пересчёта;
- диагностическое представление;
- unit и integration сценарии;
- предполагаемый performance budget.

Если отдельного состояния, инвариантов или самостоятельной ответственности нет, новый модуль, скорее всего, не нужен.

## Создание проекта

1. Создать проект в соответствующем слое `src/`.
2. Добавить проект в `Dig.sln`.
3. Добавить его в `allowed_project_references` файла `tools/quality/quality_config.json`.
4. Указать только минимально необходимые `ProjectReference`.
5. Не добавлять API движка в Domain и Application.
6. Не создавать пустые директории «на будущее».

## Публичная граница

Публичный контракт должен:

- выражать доменные операции, а не детали хранения;
- использовать стабильные идентификаторы;
- не возвращать изменяемые внутренние коллекции;
- возвращать явный успешный или ошибочный результат;
- разделять изменение состояния и чтение;
- не создавать двустороннюю зависимость.

## Проверка реализации

Перед pull request выполнить:

```bash
python tools/quality/check_quality.py
dotnet build Dig.sln --configuration Release
dotnet test Dig.sln --configuration Release
```

Также проверить:

- состояние не продублировано в соседнем модуле;
- все изменения проходят через владельца;
- ошибки имеют стабильные коды;
- события описывают произошедший факт;
- запросы не имеют побочных эффектов;
- ручные файлы не превышают 350 строк;
- неочевидное поведение доступно диагностике;
- архитектурное решение оформлено ADR, если меняет общие правила.

## Definition of Done

Модуль готов только после выполнения общего Definition of Done из `docs/development-rules.md` и критериев соответствующего GitHub Issue.
